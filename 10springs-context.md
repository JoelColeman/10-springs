## The Project

**10 Springs** — a 10-watch mechanical watch collection built over 10 years (ages 35–45), one watch purchased each spring. Hosted as a GitHub Pages dashboard at **joelcoleman.github.io/10-springs** (repo: github.com/JoelColeman/10-springs).

**Stack:** Single-file index.html, vanilla HTML/CSS/JS, Claude API chat bar for natural language updates, GitHub Pages hosting. Claude Code handles all building and commits. Strategic conversations (like this one) are the advisor layer.

**Core purpose:** Leave behind watches his children can discover independently and find hold real value — recreating the experience Joel had when he discovered his grandfather's Bulova Accutron Spaceview was rare and valuable. Substance, not obligation.

---

## Origin Story

Started as a simple question: *what are good watches under $500 that might hold value?* Joel was re-entering wearing real watches after relying on an Apple Watch, reduced by an Oura ring. From there it evolved through vintage watch education, taste discovery, and philosophy development into a named 10-year plan. The conversation became less about buying watches and more about building a collection that tells a story.

## Collection Philosophy (Locked)

- **Mechanical only** — no quartz, no exceptions (one legacy exception: the Accutron, which is not quartz but tuning fork)
- **Every watch must clear 3 of 4 bars:**
  1. Loves the aesthetic
  2. Would wear it regularly
  3. Defensible collector narrative
  4. Strong resale floor
- Vintage watches: prioritize collectibility + originality + serviceability
- Modern watches: prioritize iconic status + long-term value holding
- Balance across: Vintage / Dress / Sport / Chrono / Gold / Weird
- Framework not contract — gut aesthetic + collector floor, nothing bought to fill a box
- The goal is recreation of the Accutron discovery experience for the next generation
- **Collection subtitle:** *"Mechanical instrument watches of the exploration age"*

---

## Taste Profile

**Strongly preferred:**
- Vintage mechanical, mid-century design language
- Black dials, blue dials, tropical brown patina
- Watches that feel like scientific instruments, cockpit gauges, lab equipment, exploration tools
- Strong dial architecture: crosshair dials, sector layouts, instrument-style scales
- Historically important movements
- Chronographs with technical/instrument presence
- Tasteful "weird" watches with serious collector credibility
- Warm metal + dark dial pairings

**Less preferred:**
- Generic luxury signaling, overly safe collector choices
- Quartz in the main collection
- Two-tone bracelets that read cheap rather than warm/luxurious
- Fashion-first or trend-first recommendations

**The single most useful predictor of future taste:** instrument watch aesthetics — watches that look like they belong in spacecraft, observatories, laboratories, control rooms, or old aviation/scientific environments.

## Prompt Conventions

### All Claude Code prompts must:
1. Start with a planning step and wait for confirmation before executing
2. End with: *"After committing, provide the direct PR merge link to merge your branch into main so I can merge with one click."*
3. Include: *"If you find yourself retrying the same approach more than twice, stop and tell me what's blocking rather than continuing to loop."*

### Technical conventions learned:
- **Large file writes:** Always use bash heredoc (`cat > index.html << 'PYEOF'`) — never sub-agents or the Write tool for files over ~400 lines. The Write tool silently drops the `content` parameter on large writes.
- **Claude API model string:** `claude-sonnet-4-20250514`
- **Chat system prompt:** Use lean JSON patch approach — pass `getDashboardState()` compact JSON (~1KB), not full innerHTML (~30KB)
- **Prototype files:** Reference `_prototypes/dashboard-mockup.html` and `_prototypes/card-fan-v2.html` in the repo for design system and animation specs — don't rebuild from scratch
